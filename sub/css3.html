[CSS 정의]

CSS는 캐스케이딩 스타일 시트(Cascading Style Sheet)의 약자이다.
(*Cascading: '폭포같은, 계속되는, 연속적인')
위에서 아래로 흐르듯이(계단식으로) 작성된 스타일 시트(수십, 수백개의 스타일을 쉽게 관리하기 위해 모아놓은 것)
css3는 버전을 의미함.

[웹 페이지에서의 CSS]

HTML은 기본 구조(뼈대)에 해당하고 CSS는 사이트에 전반적인 스타일을 입히는 작업을 수행한다.

[CSS 적용하기]

1. 인라인 스타일(Inline style)

- HTML 내부 태그에 style 속성을 사용하여 CSS 속성값을 직접 입력하는 방식
- 해당 태그에만 스타일이 적용됨.
- 여러 태그에 같은 스타일을 적용하고자 할 때 번거로움이 있어 꼭 필요한 경우가 아니라면 다른 방식으로 스타일을 적용하는 것을 권장.


2. 내부 스타일 시트(Internal style sheet, Embedded Style)

- 인라인 스타일과 마찬가지로 HTML 문서 내부에 CSS 코드를 작성하는 방식
- head 내부에 <style></style>태그를 추가하여 그 사이에 스타일을 작성.
- 해당 HTML 문서에만 스타일을 적용할 수 있어 많은 페이지가 있는 경우에는 따로 작성을 해야 해서 불편함.


3. 외부 스타일 시트(External style sheet)

- 스타일 시트를 별도의 파일로 저장한 다음, 필요할 때마다 HTML 문서와 스타일 시트를 연결하여 사용하는 방식 (병렬 로딩 방식)
- 파일확장자 .css - 여러 개의 스타일 시트를 생성해서 웹페이지에 사용된 스타일 선언문을 관리할 수 있어 가장 많이 쓰는 적용 방식. - HTML의 <link> 태그를 사용하여 외부 스타일 시트 파일을 연결함.


4. Import

- 스타일 시트 내에서 다른 스타일 시트 파일을 불러오는 방식 - 여러 스타일 시트를 import방식으로 연결할 경우 순차적으로 css 파일을 불러오기 때문에 페이지 로딩 속도에 영향을 줄 수 있어 권장하지는 않음. (직렬 로딩 방식)


[CSS 문법]

- CSS는 선택자(selector)와 선언부(declaration block)로 구분된다.
- CSS는 선택자 사이에 들여쓰기를 하지 않는다.



선택자는 스타일을 적용하고자 하는 HTML 태그를 가르킨다.
선언부 시작과 끝은 중괄호({})를 사용하고 선언과 선언사이는 세미콜론(;)으로 구분한다.
마지막 선언뒤에 세미콜론은 필수는 아니지만 사용하는 것을 권장한다.

CSS 검사기 : https://jigsaw.w3.org/css-validator/

1. CSS 선택자 유형
전체 선택자 - 문서 내 모든 태그(요소)를 선택.
- *(asterisk) 기호로 문서 내에 모든 요소를 선택할 수 있다.

*{color:#ffff00;}

요소(태그) 선택자 - 태그명으로 선택.

h1{color:yellow;}
p{color:yellow;}

아이디(ID) 선택자 - 특정 영역에 지정된 id 속성값을 토대로 요소를 선택.
- html상에서는 id 속성으로 css상에서는 #(해시태그)로 구분. - 문서 내에서 한번만 사용 가능 (고유한 식별자- 구조 요소에 적용:header,main,footer등)

<div id="gnb">..</div>
#gnb{font-size:25px; color:red}
div#gnb{font-size:25px; color:red}

네이밍 규칙

1. 간결함: 되도록 짧게
2. 명료함: 분명한 말뜻(Semantics)
3. 포괄성: 대부분의 사이트에서도 적용되도록 (gnb,lnb,snb 등 공통으로 쓰이는 자주 쓰이는 것들은 이름이 포괄적이면 사용하기가 용이함) 4. 네이밍 조합:형태-의미-순서-상태 순으로 (btn_)


한글 안됨, 숫자로 시작 안됨, 공백 안됨.
영문 대소문자 구분 + 숫자 + _(언더바),-(하이픈)

하이픈(-),언더스코어(_)는 2개 이상의 단어를 조합할 때만 사용.
하이픈(-),언더스코어(_)를 이용하여 3단계를 초과하여 사용하는 방법은 지양

클래스(Class) 선택자 - 지정된 class 속성값을 토대로 요소를 선택.
- html상에서는 class 속성으로 css상에서는 .(마침표)로 구분. - 문서 내에 여러 번 중복으로 사용 가능.

<p class="menu">..</p>
.menu{font-size:25px; color:red}
p.menu{font-size:25px; color:red}

다중 클래스
- 공백으로 구분하여 여러 개의 class값을 적용시킬 수 있음. (but 3개이상 권장안함.)

<p class="menu bg">..</p>

주로 사용되는 아이디 네임과 클래스 네임 (약속어)
그룹(group) 선택자 - 여러 선택자들을 함께 사용할 때.
- 여러 선택자를 ,(쉼표)로 구분.

h1{font-size:25px; color:red}
p{font-size:25px; color:red}
.menu{font-size:25px; color:red}
h1,p,.menu{font-size:25px; color:red}

하위(자손) 선택자 상위요소에 포함된 모든 하위 요소에 스타일 적용. 상위요소와 하위요소 사이에 공백

p span {font-size:25px;}

자식 선택자 부모요소 바로 1단계 하위의 자식 요소에만 스타일 적용. 상위요소와 하위요소 사이에 부등호로 표기

#header>ul>li { color:#f00; }

종속 선택자 태그 선택자와 id선택자,class선택자가 결합된 형태.

header#header {height:1000px }

일반 형제 선택자 첫번째와 두번째 요소 사이에 ~ (틸드)를 표시함

h1 ~ p { color:#f00; }

인접 형제 선택자 첫번째와 두번째 요소 사이에 ~ (틸드)를 표시함

h1 + p { color:#f00; }

속성 선택자 특정 속성이나 특정 속성값을 가지고 있는 HTML 요소를 선택하여 스타일 적용.

- [속성명] 선택자
- [속성명="속성값"] 선택자

a[href] {baground-color:#00f}

기본형 스타일 적용 범위 문법 예시
[속성명] 지정한 속성 일 때 [href], [class], [id] a[href]{color:#f00}
[속성명="속성값"] 속성값이 일치할 때 [target="_blank"], [type="text"] input[type="text"]{color:#f00}
[속성명 ~="속성값"] 여러 속성값 중에 해당 값이 포함 되어 있을 때(일부만 일치하는 것이 아닌 한 단어로 일치해야 함) [class ~="button"] (예:flat-button, buttons 이면 해당 안됨) li[class ~="menu"]{color:#f00}
[속성명 |="button"] 특정 속성의 속성값이 지정한 값이거나 값(하이픈)-으로 시작하는 요소를 모두 선택 [class |="button"] li[class |="button"] {color:#f00}
[속성명 ^="button"] 특정 속성값으로 시작하는 요소를 모두 선택(한 단어가 아니어도 됨) [class ^="button"] li[class ^="button"] {color:#f00}
[속성명 $="last"] 특정 속성값으로 끝나는 요소를 모두 선택 [class $="last"] li[class $="last"] {color:#f00}
[속성명 *="w3"] 특정 속성값의 일부가 일치하는 요소를 모두 선택 [class *="w3"] li[class *="w3"] {color:#f00}
가상 선택자
가상(의사) 클래스[영어:슈도(pseudo) 클래스(classes)] html상에 실제로 존재하는 요소(태그)에 클래스 추가 없이 특정 조건의 상태를 선언하여 스타일을 적용할 수 있다.
구문: selector:pseudo-class {
property: value;
}
가상(의사) 요소[영어:슈도(pseudo) 클래스(elements)] html상에 실제로 존재하지 않는 요소(태그)를 생성하여 스타일을 적용할 수 있다.
구문: selector:pseudo-element {
property: value;
}
사용자 동작에 반응하는(동적) 가상(의사) 클래스

:link - 방문하지 않은 링크에 스타일 적용. 기본 스타일: 파란색 글자와 밑줄 표시
:visited - 한 번이상 방문한 링크에 스타일 적용. 기본 스타일: 자주색글자
:active - 마우스를 누르고 있을 때 스타일 적용.
:hover - 마우스 커서를 올려 놓았을때 스타일 적용.
롤오버효과 가능(이미지 위로 마우스 커서를 올리면 다른 이미지로 바뀌었다가 마우스 커서를 내리면 원래 이미지로 돌아오는 효과)
:focus - 해당 (입력)요소에 초점이 맞춰졌을때. 주로 Form요소에서 텍스트 필드 안의 커서가 활성화 되거나 tab을 눌러 초점이 이동했을 때 적용된다.

선택자 작성 순서에 유의해야 함.
:link->:visited->:hover->:active (:focus를 사용한다면 :active보다 먼저 선언)

구조 가상(의사) 클래스 웹 문서 구조를 기준으로 특정 위치에 있는 요소를 찾아 스타일을 적용

:root - 문서안의 루트 요소에 스타일 적용할 때

:nth-child(n) - 앞에서 n번째 순서에 해당하는 태그를 선택하여 CSS 적용
:nth-last-child(n) - 끝에서 n번째 순서에 해당하는 태그를 선택하여 CSS 적용
:first-child, :last-child - 첫번째, 마지막 순서에 해당하는 태그를 선택하여 CSS 적용
:nth-child(2n+1), :nth-child(odd) - 홀수번째 순서에 해당하는 태그를 선택하여 CSS 적용
:nth-child(2n), :nth-child(even) - 짝수번째 순서에 해당하는 태그를 선택하여 CSS 적용
:nth-child(n+5), :nth-child(-n+5) - 5번째부터, 5번째까지 해당하는 태그를 선택하여 CSS 적용
:nth-child(n+3):nth-child(-n+8) - 3번째부터 8번째까지 해당하는 태그를 선택하여 CSS 적용

:nth-of-type(n) - 특정 태그의 n번째 순서에 해당하는 태그를 선택하여 CSS 적용
:nth-last-of-type(n) - 특정 태그의 끝에서 n번째 순서에 해당하는 태그를 선택하여 CSS 적용
:first-of-type, :last-of-type - 첫번째, 마지막 순서에 해당하는 특정 태그를 선택하여 CSS 적용
:only-child -부모 태그 안에 자식 요소가 유일하게 하나일 때 해당 태그에 CSS 적용(다른 자식 요소 있으면 안됨.)
:only-of-type - 부모 태그 안에 해당 요소가 유일한 태그 일 때 css 적용(다른 자식 요소 있어도 됨.)
요소 상태에 따른 가상(의사) 클래스

:checked - input 요소 중에서 체크된(checked) 상태의 input 요소를 선택
:disabled - input 요소 중에서 사용할 수 없는 input 요소를 선택
:enabled - input 요소 중에서 사용할 수 있는 input 요소를 선택


input:checked + label{font-size:12px;color:#f00}
<input type="text" checked>

기타 가상 클래스(슈도(pseudo class) 클래스)

:not - 특정 선택자가 아닐 때 (예: div:not(.menu){color:#f00}) :lang - 특정 HTML 요소를 사용자 컴퓨터의 언어 설정에 따라 다르게 표현할 때


span:lang(en){font-size:15px;color:#ddd}

2. CSS 선언 우선순위(css 명시도)

css 선언 중요도 순서
1. 사용자 스타일 시트 (저시력자나 색약자 등과 같이 특별한 조건이 필요한 사용자들에게 맞게 구성해 놓은 시스템을 통해 만들어진 스타일 시트:제작자 제어할 수 없음)
2. 제작자 스타일 시트
3. 기본적인 브라우저 스타일 시트 (브라우저에서 HTML태그를 해석하여 보여주는 기본 타입)

- 스타일시트는 여러 개의 스타일 선언문으로 구성되어 있다.
- 스타일 선언문에서 선택자가 겹치는 경우 브라우저는 우선순위(명시도)에 따라 스타일을 적용한다.

명시도란 브라우저가 어느 요소와 가장 연관된 속성을 찾는 수단으로, 이렇게 찾은 속성이 해당 요소에 적용된다.
명시도는 여러 종류의 CSS 선택자로 구성된 일치 규칙에 기반한다.


1. 선택자 우선순위를 계산하여 값이 높은 순서대로 적용한다.
2. (순위가 같다면) 가장 마지막에 선언한 선택자의 스타일을 우선적으로 적용한다.


- 4자리 숫자로 표현 후 숫자가 큰 순으로 우선 순위 결정


각 자리의 숫자 결정방법
무한대 - 속성값 뒤에 !important 를 붙임.(세미콜론 닫히기 전에 작성해야 함.) - 어떤 스타일보다 가장 우선으로 적용됨.

p{font-size:35px !important} p.name{font-size:25px}

인라인 스타일
(1000점) 		- 태그 안에 style 속성을 사용하여 적용된 스타일

<p style="font-size:50px">..</p>
p{font-size:35px}
p.name{font-size:35px}

id 선택자
(100점) 		- 특정 영역에 해당하는 하나의 요소(태그)에 적용된 스타일

p#name{font-size:50px}
p.name{font-size:35px}

class 선택자
(10점) 		- class 속성값을 가진 요소(태그)에 적용된 스타일

p#name{font-size:50px}
p.name{font-size:35px}

태그(요소) 선택자
(1점) 		- 요소(태그)를 선택하여 적용된 스타일

p{font-size:50px}
p.name{font-size:35px}

전체 선택자
(0점) 		- 전체 요소(태그)를 선택하여 적용된 스타일

*{font-size:50px}
p{font-size:35px}

1. 폰트와 관련된 속성
font-family - 글꼴을 설정하는 속성.
- 기본 글꼴 (윈도우즈10 크롬&엣지: 맑은 고딕,맥OS사파리:Apple SD 산돌고딕 Neo)
- 자손에게 상속되는 속성.
- 한 개 이상의 글꼴을 지정할 수 있고 ,(쉼표)로 구분함.
- 가장 먼저 쓰여진 글꼴부터 우선 순위로 적용.
- 한글이나 두 단어로 조합된 영단어일 경우 따옴표 묶어주어야 함.

선택자{font-family: 글꼴이름1, 글꼴이름2, …,;}
html{font-family:Arial,"Times New Roman","malgun gothic'}

generic-family - 비슷한 모양을 가지는 글꼴을 설정하는 속성값.
- font-family 속성값 맨 뒤에 작성함.(키워드이기 때문에 따옴표로 묶지 않음)
- serif: 삐침 있는 글자
- sans-serif: 삐침 없는 글자
- monospace: 글자 폭이 동일한 글자
- cursive: 손글씨 느낌의 글자(흘림체)

html{font-family:Arial,"Times New Roman","malgun-gothic',sans-serif}

한번에 여러 나라 언어 설정 시(다국어 폰트 동시 적용), 영어, 일본어, 한국어 순으로 폰트 나열해야 함.
font-style - 글자 스타일 선언.

p{font-style:italic;}

속성값 내용
normal

기본값으로, 글자를 일반적인 형태로 표시
italic

글자를 이탤릭체로 표시.
oblique

글자를 기울여서 표시(이탤릭과 유사하지만 기울임 정도를 지정할 수 있지만 css4를 지원하는 브라우저에서만 사용가능)

font-weight oblique <각도> (기본 각도 14도. 유효한 값은 -90 ~ 90도)
참고사이트 : MDN

font-size - 폰트 크기를 설정
- 기본값은 16px
- 폰트 크기를 지정할 때는 단위와 함께 입력해야 함.

폰트 단위에 사용하는 단위 종류 (절대단위 vs 상대단위)

절대단위 - px: 화면을 구성하는 최소 단위인 픽셀을 뜻함.
- 어떤 단위를 사용하더라도 최종적으로는 px로 계산된 값을 브라우저가 사용함.

h2{font-size:20px;}

상대단위 - % : 상위 요소의 폰트 크기를 기준으로 백분율로 계산하여 적용 (1em=100%=16px)
- em : 해당 폰트(font)의 기본 크기를 1em으로 놓고, 그에 대한 상대적인 크기를 설정 (부모 요소의 font-size에 em 값을 곱한 크기)
- rem : 루트 요소의 폰트 크기를 배수로 계산하여 적용
- root 요소(태그)는 문서의 시작 부분을 뜻함.
- root 요소(태그)에 폰트 사이즈 기본 값 설정 시 % 단위로 설정하는 것을 권장.(이유: 사용자의 접근성 옵션)
- viewport units (vw,vh)
- vw : 뷰포트 가로를 기준으로 계산. (1vw = width의 1%)
- vh : 뷰포트 세로를 기준으로 계산. (1vh = height의 1%)

예: 뷰포트 가로가 1000px일 때 1vw=10px, 100vw=1000px
p{font-size:10vw;}

그 밖에 font-size 속성에 입력할 수 있는 키워드

- medium : 웹브라우저에서 정한 기본 글자크기.
- xx-small, x-small, small, large, x-large, xx-large : medium에 대한 상대적인 크기.
- smaller, larger : 부모 요소의 글자 크기에 대한 상대적인 글자 크기.

larger : 부모값 + 부모값 * 1/5
(ex) 부모 요소의 폰트 크기가 20px 일 때 20px + 4px 하여 24px이 된다.

smaller : 부모 - 부모 * 1/6
(ex) 부모 요소의 폰트 크기가 30px 일 때 30px - 5px 하여 25px이 된다.
font-weight - 글자 굵기를 설정

p{font-weight:bold}

normal 기본 값(400)
bold 굵게 표현(700)
bolder 부모 요소 보다 두껍게 표현
lighter 부모 요소 보다 얇게 표현
number 100, 200, 300, 400, 500, 600, 700, 800, 900 (숫자가 클수록 두껍게 표현)

일반적으로 자주 사용하는 굵기는 넘버를 사용하여 설정함.
폰트가 normal과 bold만 지원하는 경우에는 넘버를 나눠 사용하더라도 크게 변화가 없음.

2. 웹폰트 적용하기

웹 폰트: 웹 사이트 이용자가 로컬 컴퓨터에 폰트가 설치되어 있지 않은 경우에도 온라인의 특정 서버에 위치한 폰트를 다운받아 화면에 표시해주는 ‘웹 전용 글꼴’

웹 폰트 확장자
  • 1. .woff(Web Open Font Format) : 기본적으로 TrueType이나 OpenType 서체를 압축한 것이기 때문에 일반적으로 파일 크기가 작아서, 다른 서체보다 신속히 다운로드 됨
  • 2. .woff2: woff2 형식이 30~50% 더 압축된 형식
  • 3. .eot: ie 전용폰트
  • 4. .svg: SVG(크기 조정이 가능한 벡터 그래픽)아트워크로 표시되는 글리프 전체 또는 일부가 포함된 글꼴 형식.
적용방법
  • 1. .ttf, .otf 파일을 웹폰트로 변환한 뒤 서버에 저장시킨 웹 폰트 적용.
  • 2. CDN(Content Delivery Network) 방식으로 URL을 이용하여 웹 폰트 적용
    * 지리적 제약 없이 전 세계 사용자에게 빠르고 안전하게 콘텐츠를 전송할 수 있는 콘텐츠 전송 기술을 의미함.
    * CDN방식은 서버와 사용자 사이의 물리적인 거리를 줄여 콘텐츠 로딩에 소요되는 시간을 최소화할 수 있다는 장점이 있음.
1. 구글 폰트 사이트 이용하기
- 구글 폰트 웹사이트에서 폰트를 검색하고 사용하고자 하는 폰트 굵기 등을 선택한 뒤 링크를 복사해서 내 문서에 붙여넣기.
- 영문 폰트 사용시 가장 많이 이용하고 국문 폰트 종류도 조금 있어 이용하기에 편함.
2. 서버에 폰트 업로드 한 후 적용하기
- .ttf, .otf 폰트 파일을 웹폰트로 변환한 후 변환된 폰트를 서버에 업로드 함.
- css에 폰트 선언하기. (format() 반드시 사용:브라우저에서 지원 가능한 파일만 다운로드 받을 수 있게 한다.)
- local 폰트를 사용하여 해당 폰트가 시스템에 설치되어 있다면 리소스를 요청하지 않도록 한다.
- woff2를 가장 앞에 선언하여 브라우저가 먼저 다운받도록 한다.

- font-display 속성을 이용하면 웹 폰트의 로딩 상태에 따른 동작을 설정할 수 있다.
- font-display 속성은 auto, block, swap, fallback,optional의 5가지 옵션이 있다.

auto : 브라우저의 기본 동작
block : FOIT 방식
swap : FOUT 방식
fallback : 100ms동안 텍스트가 보이지 않고 그 후 fallback 폰트로 렌더링한다. 특이한 점은 약 2초의 전환(swap)시간이 있다는 점이다. 이 시간 안에 로딩이 완료되면 웹 폰트로 전환한다
optional : fallback과 비슷하지만 다르게 작동한다. 네트워크 상태를 파악하여 웹 폰트 전환 여부를 판단.

속성값 설명
normal 기본값. (CJK를 제외한 비아시아권 언어: 텍스트가 길고 중단점(공백,하이픈)이 없으면 줄바꿈 안 함)
break-all 텍스트가 문자(글자 하나하나)단위로 잘려 아래로 줄바꿈.
keep-all 텍스트에서 단어 단위로 잘려 아래로 줄바꿈.(중단점: 공백, 하이픈, CJK는 그 외 기호)
initial 초기화
inherit 부모 요소의 속성값을 상속
text-shadow - 텍스트에 그림자 효과 추가하기

h2 { text-shadow: 2px 1px #3399CC; }

속성값 설명
가로 거리(X좌표) 텍스트부터 그림자까지의 가로 거리. 양수값은 글자 오른쪽, 음수값은 글자 왼쪽에 그림자 만들어짐. 필수 속성
세로 거리(Y좌표) 텍스트부터 그림자까지의 세로 거리. 양수값은 글자 아래쪽, 음수값은 글자 위쪽에 그림자 만들어짐. 필수 속성
번짐정도(blur) 그림자 번짐 정도. 기본값은 0. 양수값은 그림자가 사방으로 퍼져나가고 음수값은 모든 방향으로 축소됨.
색상 그림자 색상. 기본값은 글자색. 색상명, rgb, 헥스코드로 표현가능
4. 배경(background)과 관련된 속성
background-color - 요소의 배경 색상을 지정하는 속성.
- 기본값: 투명 (transparent)

선택자{background-color: yellow;}
html{background-color: yellow;}
p {background-color: #FFFFCC;}
- 그라디언트(gradient):둘 이상의 색 사이의 색상 표현을 부드럽게 전환해주는 효과를 의미
  • 1. 선형 그래디언트(linear gradients): 색상이 수직이나 수평 또는 대각선 방향으로 일정하게 변하는 것.

    linear-gradient(각도 혹은 진행방향, color-stop, color-stop, ...);

    background-image: linear-gradient(to right, red 0%, yellow 50%, green 100%);
    혹은
    background: linear-gradient(to right, red, orange, yellow, green, blue, indigo, purple);
    background: linear-gradient(left, red, orange, yellow, green, blue, indigo, purple);
    background: linear-gradient(left bottom, red, orange, yellow, green, blue, indigo, purple);
    background: linear-gradient(to right top, red, orange, yellow, green, blue, indigo, purple);
    background: linear-gradient(135deg, orange 20%, cyan); background: linear-gradient(45deg, red, blue);


    to bottom : 위에서 아래로 그라데이션을 만든다. (기본값)
    to top : 아래에서 위로 그라데이션을 만든다.
    to left : 오른쪽에서 왼쪽으로 그라데이션을 만든다.
    to right : 왼쪽에서 오른쪽으로 그라데이션을 만든다.
    ndeg : n도의 방향으로 그라데이션을 만든다.

  • 2. 원형 그래디언트(radial gradients): 색상이 원이나 타원의 중심부터 바깥 방향으로 변하는 것.
    radial-gradient(모양 및 크기 at 중심점, color-stop, color-stop, ...);

    1. 모양 : circle, ellipse(기본값)
    2. 크기
    - closest-side : 원형 그래디언트의 크기가 가장 가까운 면에 닿을 만큼의 크기로 설정
    - farthest-side : 원형 그래디언트의 크기가 가장 먼 면에 닿을 만큼의 크기로 설정
    따라서 가까운 면에서는 그래디언트의 일부분이 화면을 넘음.
    - closest-corner : 원형 그래디언트의 크기가 가장 가까운 모서리에 닿을 만큼의 크기로 설정
    - farthest-corner : 원형 그래디언트의 크기가 가장 먼 모서리에 닿을 만큼의 크기로 설정
    이 크기가 기본 설정이며, 가까운 모서리에서는 그래디언트의 일부분이 화면을 넘음.

8. 박스 모델(box-model)

- 박스 모델(box-model)이란, HTML 문서를 이루는 각각의 요소들을 박스 형태로 구성하여 브라우저에 배치하기 위한 일종의 디자인 모델을 말하며 간단하게 HTML 요소들을 감싸는 상자라고 볼 수 있다.
- CSS로 스타일을 정의할 수 있는 모든 요소에는 박스 모델 관련 속성 중 일부 또는 전부를 적용할 수 있으며, CSS는 영역 별로 크기를 따로 설정할 수 있도록 다양한 속성들을 제공한다.

1. 박스 모델(box-model)의 종류에는 한 줄에 하나의 요소만 표시되는 블록 레벨 요소와, 한 줄에 여러 개의 요소가 표시되는 인라인 레벨 요소가 있다.
2. 박스 모델(box-model)은 content(콘텐츠), padding(안쪽 여백), border(테두리), margin(바깥 여백) 으로 구성되어 있다.

content 요소 내에 포함된 콘텐츠를 표시하는 영역으로, 관련 속성은 width, min-width, max-width, height, min-height, max-height
padding 콘텐츠와 테두리 사이의 간격.(안쪽 여백)
margin - 요소와 요소 간의 여백(바깥 여백)
- 경계선 바깥을 감싸는 여백(바깥 여백)
border - 콘텐츠와 패딩 주변을 감싸는 경계선
- margin과 padding의 경계
9. 콘텐츠 영역의 크기
width - 요소의 너비 설정
- 블록 레벨 요소(인라인 블록 레벨 포함)를 가진 요소에만 적용 (예외: flexbox item, grid item일 때는 인라인 레벨 요소도 width값 가질 수 있음)
- %, em, px, vw, vh 단위 사용 가능 (공통사항: 값이 0일 경우에는 단위 생략 가능)
auto - 브라우저가 요소의 너비에 맞게 자동으로 너비를 지정함. 기본값
- 가용 너비에서 여백(margin)을 제외한 너비를 요소의 width 속성값
% 부모 요소의 width에 상대적인 크기를 지정함
em 해당 요소의 font-size에 대한 비율(1em=16px)로 지정함
px 고정값으로 지정.
vw, vh - vw : 뷰포트 가로를 기준으로 계산. (1vw = width의 1%)
- vh : 뷰포트 세로를 기준으로 계산. (1vh = height의 1%)
min-content - 최소 너비는 해당 요소가 담고 있는 컨텐츠에 의해 width 속성값이 결정
- 요소의 너비를 줄일 수 있는 만큼 최소로 줄이고 싶을 때 사용
- 텍스트 컨텐츠의 경우 최대한 줄바꿈을 하여 가장 긴 단어의 길이가 최소 너비를 결정
max-content - 최대 너비는 해당 요소가 담고 있는 컨텐츠에 의해 width 속성값이 결정
- 요소의 너비를 최대한 늘이고 싶을 때 사용
- 텍스트 컨텐츠의 경우 줄바꾸기를 하지 않은 상태의 문장의 길이가 최대 너비를 결정
- 텍스트의 배경색을 적용할 때 유용하게 사용
fit-content - max-content 키워드와 auto 키워드의 중간
- 가용 너비가 부족하지 않는 경우에는 max-content 키워드처럼 담고 있는 컨텐츠의 최대 너비를 요소의 width 속성값
- 가용 너비가 부족하면 auto 키워드처럼 가용 너비에서 여백(margin)을 제외한 너비를 요소의 width 속성값으로 사용
initial 초기값으로 지정.
inherit 부모로부터 상속받은 값

div{width:500px;}

min-width - 요소가 가질 수 있는 최소 너비(width)를 설정
- 요소의 width값이 min-width값보다 작아지는 것을 방지함.
- 기본값은 0
- min-width는 width와 max-width보다 우선

div{width:100%; min-width:300px}

max-width - 요소가 가질 수 있는 최대 너비(width)를 설정
- 기본값은 none

div{width:500px; max-width:300px}

height - 요소의 높이 설정
- 블록 레벨 요소를 가진 요소에만 적용(인라인 블록 레벨 포함)
- %, em, px, vw, vh 단위 사용 가능
auto 브라우저가 요소의 높이에 맞게 자동으로 높이를 지정함. 기본값
% 부모 요소의 height에 상대적인 크기를 지정함.
em 해당 요소의 font-size에 대한 비율(1em=16px)로 지정함
px 고정값으로 지정.
vw, vh - vw : 뷰포트 가로를 기준으로 계산. (1vw = width의 1%)
- vh : 뷰포트 세로를 기준으로 계산. (1vh = height의 1%)
initial 초기값으로 지정.
inherit 부모로부터 상속받은 값

div{height:500px;}

min-height - 요소가 가질 수 있는 최소 높이(height)를 설정
- 요소의 height값이 min-height값보다 작아지는 것을 방지함.
- 기본값은 0

div{min-height:300px}

max-height - 요소가 가질 수 있는 최대 높이(height)를 설정
- 기본값은 none

div{height:500px; max-height:300px}

box-sizing

- 요소의 너비와 높이를 계산하는 방법을 지정함.
- width와 height가 박스 모델의 영역(padding, border) 중 어느 영역까지의 크기를 기준으로 값을 가지게 될 것인지를 결정하는 속성
- margin 값은 width 값에 포함되지 않음

속성값 설명
content-box 기본값, width와 height 속성이 콘텐츠 영역만 포함하고 안팎 여백과 테두리는 포함하지 않음
너비 = 콘텐츠 너비, 높이 = 콘텐츠 높이로 계산
border-box width 와 height 속성이 안쪽 여백과 테두리는 포함하고, 바깥 여백은 포함하지 않음
너비 = 테두리 + 안쪽 여백 + 콘텐츠 너비, 높이 = 테두리 + 안쪽 여백 + 콘텐츠 높이로 계산함.
10. overflow

- 자식 요소 내용이 부모 요소의 범위를 초과할 때, 요소 내용을 어떻게 처리할지 결정할 때 사용.

- 부모 요소의 높이값이 지정되어 있을 때 부모요소에 overflow 속성 값 적용함.

visible - 기본값
- 자식 요소의 내용이 박스를 넘어 가더라도, 그대로 보여줌.

div{height:500px; overflow:visible;}

hidden - 부모요소의 범위를 넘어가는 자식요소의 내용은 보이지 않음

div{height:500px; overflow:hidden;}

scroll - 부모요소의 범위를 넘어가는 자식요소의 내용은 보이지 않고 사용자가 내용을 확인 할 수 있도록 스크롤바를 표시함.
auto - 부모요소의 범위를 넘어가는 자식요소의 내용이 있을 경우, 해당 부분을 보이지 않도록 처리하고, 사용자가 내용을 확인 할 수 있도록 스크롤바를 표시함.
overflow-x 와 overflow-y - 가로 세로 부분을 각각 따로 지정할 때 사용함.

div{height:500px; overflow-x:hidden; overflow-y:scroll;}

11. 여백
padding - 콘텐츠와 테두리 사이의 간격.(안쪽 여백)
- 인라인 레벨 요소의 경우 좌우 여백은 적용되나 상하 여백은 적용되지 않음
- %, em, px 단위 사용 가능

속성 설명
padding 모든 padding 속성을 한 번에 선언.
padding-top 윗쪽(상)의 패딩(padding)값을 설정함.
padding-right 오른쪽(우)의 패딩(padding)값을 설정함.
padding-bottom 아래쪽(하)의 패딩(padding)값을 설정함.
padding-left 왼쪽(좌)의 패딩(padding)값을 설정함.

div{padding-top:50px; padding-left:30px;}

margin - 요소와 요소 간의 여백(바깥 여백)
- 경계선 바깥을 감싸는 여백(바깥 여백)
- 인라인 레벨 요소의 경우 좌우 여백은 적용되나 상하 여백은 적용되지 않음
- %, em, px 단위 및 auto 속성값 사용 가능
- 음수 사용 가능 (해당 요소를 다른 요소 위에 겹치게 할 때 사용함.)

속성 설명
margin 모든 margin 속성을 한 번에 선언.
margin-top 윗쪽(상)의 마진(margin)값을 설정함.
margin-right 오른쪽(우)의 마진(margin)값을 설정함.
margin-bottom 아래쪽(하)의 마진(margin)값을 설정함.
margin-left 왼쪽(좌)의 마진(margin)값을 설정함.

div{margin-bottom:50px; margin-left:30px;}

margin 좌우값 auto - 브라우저가 수평 방향 마진(margin)값을 자동으로 설정.
- 지정된 width 값 만큼 공간을 차지하고 남은 공간을 균등하게 나눠가짐.
- 수평 중앙 정렬을 하기 위해서는 해당 요소에 width값이 선언되어 있어야 함.
margin 중첩현상 - 마진 중첩(margin overlap) 또는 마진 병합(margin collapse)라고 함
- 두 요소가 상하로 정렬된 경우: 위 요소의 하단 마진과 아래 요소의 상단 마진이 겹칠 때
- 겹쳐질 경우 마진값이 큰 쪽으로 겹쳐짐
- 수평으로 정렬될 때는 해당없음.

부모 요소와 첫 번째 자식 요소 또는 마지막 자식 요소

- 부모 요소의 상단 마진과 첫 번째 자식 요소의 상단 마진 병합이 일어남.
- 부모 요소의 하단 마진과 마지막 자식 요소의 하단 마진 병합이 일어남.
- 해결방법:부모요소(상단 혹은 하단)에 보더 또는 패딩 값이 있으면 마진 병합이 일어나지 않음

내용이 없는 빈 요소의 경우

- 해당 요소의 상단 마진과 하단 마진의 병합이 일어남
- 마진 병합은 인접한 두 박스가 온전한 block-level요소일 경우에만 적용됨.(inline-block 등의 요소일 때는 해당 안됨)
- 수평으로 정렬될 때는 해당없음.
마진 병합 예외인 경우 - 박스가 position: absolute 된 상태
- 박스가 float: left/right 된 상태 (단, clear 되지 않은 상태)
- 박스가 display: flex 일 때 내부 flexbox item
- 박스가 display: grid 일 때 내부 grid item

padding, margin 속성 각각 한 번에 선언하기(축약)

div{padding:50px; margin:30px;}

속성값 한 개일 때 네 방향에 동일한 값 설정
속성값 두 개일 때 상하 여백과 좌우 여백의 값을 각각 따로 지정
속성값 세 개일 때 윗쪽 여백, 좌우 여백, 아래쪽 여백 순서로 값을 지정
속성값 네 개일 때 윗쪽, 오른쪽, 아래쪽, 왼쪽 순서대로 각각을 따로 지정
12. box-shadow

- 요소(태그)에 그림자 효과를 만들어주는 속성

box-shadow: 그림자의 x축 값 그림자의 y축 값 blur값 spread값 색상값 inset;

속성값 설명
x-position 그림자 가로 위치. 양수면 오른쪽에, 음수면 왼쪽에 그림자가 만들어짐(필수값)
y-position 그림자 세로 위치. 양수면 아래쪽에, 음수면 위쪽에 그림자가 만들어짐(필수값)
blur 그림자를 흐릿하게 만든다. 값이 클 수록 더욱 흐려짐
spread 양수면 그림자를 확장하고, 음수면 축소.(그림자의 크기)
color 그림자 색 지정
inset 그림자가 요소의 바깥쪽이 아닌 안쪽에 만들어짐
none 그림자 효과를 없앰(기본값)
initial 초기값으로 설정
inherit 부모 요소의 값을 상속받음

한꺼번에 여러개의 그림자를 생성할 때는 각 그림자를 콤마(,)로 구분함.

div { box-shadow:5px 5px gray,10px 10px 10px yellow; }

13. opacity

- 요소의 불투명도를 설정
- 기본값 1 또는 100% (허용값:0.0 ~ 1.0, 0% ~ 100%)
- 숫자가 클수록 불투명하고, 숫자가 작을 수록 투명함.
- 부모가 opacity값이 있으면 자식 node들도 투명해지기 때문에 주의해야 함.

속성값 설명
number 0.0부터 1.0까지의 수, 0%~100%
initial 기본값
inherit 부모 요소의 값을 상속받음
alpha

- 요소의 불투명도를 설정
- 기본값 1 (허용값:0.0 ~ 1.0)
- 숫자가 클수록 불투명하고, 숫자가 작을 수록 투명함.
- 자식 node에 영향 주지 않음.
- RGB 색상값에 알파 채널 값을 더해 rgba로 표현함.

div { background-color:#ff0; opacity:80%; }
div{ background-color:rgba(255,255,0,0.8); }

14. display

- HTML 요소를 웹 브라우저에 어떻게 보여줄 것인지를 결정
- 태그(요소)마다 기본값이 다름.
- display 속성값에 따라 해당 요소 박스의 유형을 변경할 수 있다.
- 내부와 외부 디스플레이 유형으로 크게 나눌 수 있음.


14-1. 외부 디스플레이 타입(display-outside)
요소가 플로우 레이아웃에 참여하는 방식을 결정. 즉, 요소 자신에 적용되는 속성값.
속성값 설명
none 요소를 브라우저에 나타내지 않음. 크기도 차지하지 않음.
block 기본적으로 요소가 가로 한 줄을 다 차지함.(width:100%)
inline 요소의 너비는 컨텐츠 크기 만큼만 가짐. 줄 바꿈이 되지 않고 인라인에 요소 배치됨. 크기를 가질 수 없음.
inline-block inline과 block의 특성을 합쳐놓은 속성. 기본적으로는 inline의 속성을 지니고 있지만, 너비와 높이를 설정할 수 있음.
inline-table 인라인 레벨 요소의 표로 랜더링
인라인 레벨 요소 사이의 공백과 개행(줄바꿈)처리된 inline 요소의 경우 공백과 개행에 대해서 하나의 여백으로 인식함.
그래서 inline와 inline-block의 경우 태그 사이의 공백이나 개행이 있을 경우 약 4px의 여백을 가지게 됨.
list-item 목록의 항목처럼 랜더링 (li태그를 사용한 것처럼)
inherit 부모(상위) 요소의 값을 상속받음

유의할 점: display 속성값을 변경해도, 실제로 해당 요소가 완전히 다른 타입의 요소로 바뀌는 것은 아니므로 인라인 태그 내부에 블록 태그를 포함할 수 없음.

display와 box model의 관계
display width height margin padding border

18-1. object-fit 속성

요소의 크기에 맞게 img태그와 video태그의 크기를 조정하는 방법으로 사용되는 속성

속성값 설명
fill - 기본값
- 부모요소의 지정된 너비와 높이에 따라 요소를 가득 채울수 있는 크기로 변화되면서 비율은 유지되지 않는다. (background-size:100% 100% 느낌과 유사)
contain - 요소의 가로나 세로 크기에 가능한 딱 맞춰져 크기가 조정되고, 비율은 유지된 상태.
- 남는 공간이 발생
cover - 요소의 가로나 세로 크기에 맞춰져 꽉 채워지며 비율은 유지된 상태. (지정된 너비와 높이를 가득 채울 때까지 확대 및 축소됨)
- 잘리는 영역이 발생
none - 크기가 조정되지 않고 원본사이즈로 처리되며 크면 잘리고, 작으면 남음.
scale-down - 크기가 아무것도 지정되지 않거나 혹은 contain이 지정되어 있는 것처럼 변경됨.
- none과 contain 중 대체 콘텐츠의 크기가 더 작아지는 값을 선택
- 원본 크기보다 작아지는 결과를 보여줌.
18-2. object-position 속성

- object-position 속성은 img, video와 같은 대체 요소의 콘텐츠를 요소에 배치할 위치를 조절
- object-fit을 적용하면 기본적으로 요소의 가운데로 이미지(영상)를 이동시킨다.
- 기본값은 50%, 50%
- 숫자형 px, em, % 등이 사용되며, 키워드 top, left, right, bottom이 사용될 수 있다.
- 상속됨

속성값 설명
fill - 기본값
- 부모요소의 지정된 너비와 높이에 따라 요소를 가득 채울수 있는 크기로 변화되면서 비율은 유지되지 않는다. (background-size:100% 100% 느낌과 유사)
contain - 요소의 가로나 세로 크기에 가능한 딱 맞춰져 크기가 조정되고, 비율은 유지된 상태.
- 남는 공간이 발생
cover - 요소의 가로나 세로 크기에 맞춰져 꽉 채워지며 비율은 유지된 상태. (지정된 너비와 높이를 가득 채울 때까지 확대 및 축소됨)
- 잘리는 영역이 발생
none - 크기가 조정되지 않고 원본사이즈로 처리되며 크면 잘리고, 작으면 남음.
scale-down - 크기가 아무것도 지정되지 않거나 혹은 contain이 지정되어 있는 것처럼 변경됨.
- none과 contain 중 대체 콘텐츠의 크기가 더 작아지는 값을 선택
- 원본 크기보다 작아지는 결과를 보여줌.
18. transform

- 요소의 크기나 위치를 바꾸는 속성 - 요소에 2D 또는 3D 변형(transform)을 적용함.

속성값 설명
translate(x, y) - 현재 위치에서 해당 요소를 주어진 x축과 y축의 거리만큼 이동
- 양수이면 해당 축의 양의 방향으로, 음수이면 해당 축의 음의 방향으로 이동
- translateX, translateY 분리해서 사용가능
scale(x, y) 해당 요소의 크기를 주어진 배율만큼 늘리거나 줄입니다. 주어진 배율이 1보다 크면 크기를 늘리고, 0보다 크고 1보다 작으면 크기를 줄입니다. scaleX, scaleY
rotate(deg) 해당 요소를 주어진 각도만큼 시계 방향이나 반시계 방향으로 회전시킵니다. 주어진 각도가 양수이면 시계 방향으로, 음수이면 반시계 방향으로 회전시킵니다.
skew(x, y) 해당 요소를 주어진 각도만큼 x축 방향으로 기울입니다. skew() 함수에 포함된 skewX() 함수와 skewY() 함수는 각각 비틀기의 중심축을 x축과 y축으로 하는 함수입니다. x축을 중심으로 비틀기를 할 때는 요소의 양 끝이 좌우로 비틀어지고, y축을 중심으로 비틀기를 할 때는 요소의 양 끝이 위아래로 비틀어집니다.
matrix() 6개의 2D transform 메소드를 한 줄에 설정할 수 있도록 해줍니다. 6개의 매개 변수 값은 차례대로 scaleX(), skewY(), skewX(), scaleY(), translateX(), translateY() 함수 값을 의미

여러 속성값 적용 시

transform: translate(40px, 50px) scale(2,2);

transform-origin

- 변형 기준점 설정
- transform-origin: X축 Y축

속성값 설명

Y축 원점 기준의 y좌표값으로 길이 값이나 백분율, top, center, bottom 중에서 사용 가능
Z축 원점 기준의 z좌표값으로 길이 값만 사용 가능
19. transition

- 요소에 지정되어 있는 스타일 속성을 완전히 다른 스타일로 변화시킬 수 있는 속성
- 변화할 때의 시간이나 변화 속도 등을 추가로 설정할 수 있어 다양한 애니메이션 효과 구현 가능.

속성명 설명
transition-property - 요소에 추가할 전환(transition) 효과를 설정
- 변화 대상 속성이 여러 개인 경우에는 쉼표(,)로 구분하여 여러 속성값을 나열

transition-property : width, height, background-color;

transition-duration - 전환 효과가 실행될 시간을 설정
- 시간에 해당하는 값이 지정되어야 함. 초를 뜻하는 s나 밀리초를 뜻하는 ms 단위를 사용
transition-timing-function 전환(transition) 효과의 시간당 속도를 설정(실행 시간 동안의 속도 변화 방식을 지정)
속성값 설명
ease 기본값으로, 전환(transition) 효과가 천천히 시작되어, 그 다음에는 빨라지고, 마지막에는 다시 느려짐.
linear 전환(transition) 효과가 처음부터 끝까지 일정한 속도로 진행
ease-in 전환(transition) 효과가 느리게 시작하고 점점 빨라짐.
ease-out 전환(transition) 효과가 빠르게 시작했다가 점점 느려짐.
ease-in-out 전환(transition) 효과가 천천히 시작되어, 천천히 끝남.
cubic-bezier 베지에 곡선. 전환(transition) 효과 커스텀
cubic-bezier(n,n,n,n): n에 사용할 수 있는 값은 0~1임.

ease === cubic-bezier(0.25, 0.1, 0.25, 1.0) linear === cubic-bezier(0.0, 0.0, 1.0, 1.0) ease-in === cubic-bezier(0.42, 0, 1.0, 1.0) ease-out === cubic-bezier(0, 0, 0.58, 1.0) ease-in-out === cubic-bezier(0.42, 0, 0.58, 1.0)

transition-delay 전환(transition) 효과가 나타나기 전까지의 지연 시간을 설정
transition 단축 속성으로 변화 속성, 시간, 실행 곡선 등을 한 번에 지정
속성 지정 순서가 명확하게 정해져 있지는 않다. 다만 실행 시간과 지연 시간을 나타내는 값의 경우 실행 시간에 해당하는 값을 먼저 입력해야 합니다. 또한 필요한 경우 속성값 일부는 생략할 수 있습니다. 여러 개의 속성값을 입력할 때는 각 값을 하나 이상의 공백으로 구분함
쉼표(,)를 이용해 여러 너비, 높이, 배경색 등 여러 속성의 변화를 지정

transition: background-color 2s 300ms linear, width 2s 300ms linear, height 2s 300ms linear;

20. animation

- 엘리먼트에 적용되는 CSS 스타일을 다른 CSS 스타일로 부드럽게 전환시킬 때 사용.
- transition과 다르게 사용자의 action이 없어도 자동으로 동작하게 만들 수 있다.
한 가지의 상태로만 변환되는 것이 아니라 여러 상태를 거치며 변환시킬 수 있다.
트랜지션보다 더 규모가 크고 복잡하며 다양한 효과를 가지고 있기 때문에 좀 더 정밀한 구현이 가능

@keyframes 애니메이션을 재생할 각 프레임의 스타일을 정의하는 것
from 속성이나 0% 속성에 설정한 스타일에서 출발해 to 속성이나 100% 속성에 설정한 스타일로 점차 바뀌면서 애니메이션이 재생됩니다. 키프레임은 애니메이션을 적용할 요소의 animation-name을 정의하고 그 키프레임 코드블럭에 재생할 프레임별 시간비율을 작성합니다. @keyframes 에 이름이 작성되어있지 않으면 값이 할당되지 않는다. a ~ z, 0 to 9, 언더바 (_), 대시(-) 문자만 이름으로 사용 가능하다. - 첫문자로 숫자는 올 수 없다! 0% : 애니메이션의 시작 프레임이다. 100% : 애니메이션의 마지막 프레임이다. from : 0%와 의미가 같다. to : 100%와 의미가 같다. 0%와 100%의 사이에는 여러개의 중간값을 설정해 프레임을 작성할 수 있습니다. animation 중간중간의 특정 지점들을 거칠 수 있는 키프레임들을 설정함으로써 CSS 애니메이션 과정의 중간 절차를 제어할 수 있게 한다. @keyframes은 스타일 속성 변화를 프레임 별로 구분해 정의하는 특별한 모듈로, 사용자는 @keyframes를 통해 정의한 애니메이션에 식별자를 추가하고 이를 원하는 요소에 적용할 수 있습니다. 애니메이션의 각 프레임을 구분하는 방법으로는 백분율을 이용하는 방법과 키워드를 이용하는 방법이 있습니다. CSS3에서 애니메이션 효과를 사용하기 위해서는 우선 애니메이션에 대한 키 프레임(keyframe)을 정의해야 합니다. 키프레임은 순서대로 나열 가능하며, 지정된 % 의 순서대로 처리된다. 최소한 from / to 혹은 0% / 100% 같은 시간에 대한 규칙은 포함해야 한다. 키 프레임(keyframe)에는 특정한 시간에 해당 요소가 가져야 할 CSS 스타일을 명시합니다. @keyframes 규칙 안에 이렇게 CSS 스타일을 설정해 놓으면, 해당 요소의 스타일은 특정 시간까지 현재 스타일에서 설정해 놓은 새로운 스타일로 천천히 변화할 것입니다. 애니메이션 효과가 동작하기 위해서는 먼저 animation-name 속성을 이용하여 요소와 키 프레임을 연결해 주어야 합니다.

속성명 설명 속성값
animation-name 키프레임 애니메이션 이름을 지정.
특수문자를 제외한 문자열, 숫자, -,_ 를 조합해 1글자 이상.		  
animation-duration 애니메이션 재생 시간, 또는 반복 루프 1회를 도는 시간 0, 3s, 1.5s, 300ms
기본 값 0은 애니메이션 재생되지 않음.
"s"(초), "ms"(밀리초) 단위로 표현 가능.
animation-timing-function 애니메이션 진행 속도, 또는 가속 방식을 지정. 미리 설정된 속도 커브 중 한가지를 선택. linear | ease | ease-in | ease-out | ease-in-out | step-start | step-end | steps(int,start|end) | cubic-bezier(n,n,n,n) 중 한가지 선택 가능.
animation-delay 애니메이션 시작 지연 시간.
"s"(초), "ms"(밀리초) 2가지 단위 사용 가능		 0s, 0ms, 미지정: 지연 없이 즉시 애니메이션 시작.
3s: 3초 후 애니메이션 시작.
500ms: 0.5초 후 애니메이션 시작.
animation-direction 애니메이션 재생 방향 normal: 정방향 재생
reverse: 역방향 재생
alternate: 정방향과 역방향으로 한 번씩 번갈아 재생(정 방향 시작)
alternate-reverse: 역방향과 정방향으로 한 번씩 번갈아 재생(역방향 시작)
animation-iteration-count 애니메이션 반복 횟수 지정 양수: 정수로 횟수 표기. 횟수만큼 애니메이션 반복 실행 후 정지.
infinite: 무한 반복.
animation-fill-mode 애니메이션 시작 전, 종료 후 적용할 CSS 스타일을 지정. "none"은 요소의 CSS 스타일을 유지하며, 그 외에는 키프레임 애니메이션의 CSS 스타일을 유지함. none: 기본 값. 애니메이션 중이 아닌 경우, 요소의 CSS 스타일을 유지함.
forwards: 애니메이션 중이 아닌 경우, 애니메이션 마지막 키프레임의 CSS 스타일을 유지 함.
backwards: 애니메이션 중이 아닌 경우, 첫 번째 애니메이션 키프레임의 CSS 스타일을 유지 함.(지연 시간 포함)
both: 애니메이션 시작 전에는 첫 번째 애니메이션 키프레임을 유지하며, 종료 후에는 마지막 키프레임 애니메이션의 CSS 스타일을 유지함. "none" 과는 다름.
"none" 과 "backwards" 속성 값은 애니메이션 중이 아닌 경우 다른 화면을 표시할 수 있으므로 주의해야 함.
animation-play-state 애니메이션의 초기 실행 상태를 결정.
웹페이지 로딩 후 애니메이션을 자동 실행할지 여부를 선택할 수 있음.		 paused: 애니메이션이 일시 중지된 상태 유지
running: 애니메이션이 재생중인 상태.

애니메이션 속도 함수의 이해

재생 속도 함수 설명 그래프
linear 일정한 속도로 애니메이션 재생.
ease 기본 값. 애니메이션이 느리게 시작했다 빨라지고, 다시 느리게 끝남. 시작 부분에서는 더 짧은 시간에 빨라지며, 끝나는 부분에서는 더 긴 시간 동안 느려짐.
ease-in 애니메이션이 느리게 시작함.
ease-out 애니메이션이 느리게 끝남.
ease-in-out 애니메이션이 느리게 시작했다 빨라지고 마지막에는 느리게 끝남. 시작과 끝나는 부분의 속도 증가/감소가 대칭임.
step-start steps(1,start) 와 같음. 애니메이션 마지막 프레임으로 이동.  
step-end steps(1,end) 와 같음. 애니메이션 첫 프레임으로 이동.  
steps(양의정수[,start|end]) 양의 정수만큼 계단식으로 위치가 이동합니다. 스톱모션 애니메이션처럼 끊어지는 효과를 만듭니다.
옵션인 두 번째 파라메터로 "start", 또는 "end"(기본 값)를 사용할 수 있습니다. "start" 값은 첫 번째 스텝에서 애니메이션이 시작하고, "end" 값은 마지막 스텝에서 애니메이션이 끝납니다. 		 
cubic-bezier(x1,y1,x2,y2) 제어점 위치를 표시하는 파라메터 4개의 숫자를 입력해서 베지어 곡선을 직접 그리는 방식으로 재생 속도를 제어할 수 있습니다.
베지어 곡선은 2개의 제어점 사이를 통과하는 곡선을 생성합니다.
베이어 곡선 좌표는 0부터 1까지의 값을 가지며 왼쪽 아래가 (0,0), 오른쪽 위가 (1,1) 좌표를 가집니다. 좌표 영역 사이의 좌표 2개를 순서대로 표기해서 베지어 속선을 생성하게 됩니다.

21. 미디어쿼리

- css3 모듈 중 하나. - 미디어퀴리(Media Queries)는 각 미디어 매체에 따라 다른 스타일(css style)을 적용할 수 있게 만들어 줌
- @media media-type and (media-feature-rule) { /* CSS rules go here */ }

미디어 쿼리 구문의 구성 요소: 여기 코드가 어떤 미디어를 위한 것인지 브라우저에 알려주는 미디어 유형(예를 들어, 인쇄 또는 화면). 괄호로 묶은 CSS 규칙이 적용되기 위해 전달되어야 하는 규칙 또는 조건문 격인 미디어 표현식. 조건문을 통과하고 미디어 유형이 올바른 경우 적용되는 CSS 규칙 집합. 당신이 지정할 수 있는 미디어 유형 all print screen and를 사용해 미디어 기능을 결합할 수 있습니다

미디어쿼리 사용방법
  • 1. head태그 안 link 태그 요소에 사용하여 특성이 조건에 맞을때 css 파일을 불러오게 함.

    >link rel="stylesheet" media="screen and (max-width: 768px)" href="media.css"<
    /*media속성의 "screen and (max-width: 768px)"의 의미는 미디어 타입이 스크린이고, 화면의 최대 너비가 768px일때 media.css 내 스타일 적용*/

  • 2. 스타일 시트내에서 @media 를 사용.

    @media screen and (max-width: 768px) { body { background-color: lightgreen; } }
